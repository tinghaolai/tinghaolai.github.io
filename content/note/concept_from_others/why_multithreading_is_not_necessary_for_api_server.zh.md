## 為什麼 API 伺服器不需要多執行緒

多執行緒不適合用於撰寫伺服器。
多執行緒是一種過度的技術，將在未來變得越來越專業化。

### 為什麼呢？

因為容器化和叢集化。

如何利用伺服器的效能已不再是伺服器本身的問題。
而是容器和叢集的問題。
